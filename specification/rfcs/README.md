# GENEALOGIX RFCs (Request for Comments)

Major changes to the GENEALOGIX specification follow an RFC process.

## When to Write an RFC

- Add new entity types
- Change required fields
- Modify validation rules
- Add new core concepts
- Change file structure requirements

## RFC Process

1. **Draft**: Copy `0000-template.md` and write your proposal
2. **Submit**: Open a pull request with your RFC
3. **Discussion**: Community discusses in PR comments
4. **Decision**: Maintainers accept, reject, or request changes
5. **Implementation**: Accepted RFCs are merged and implemented

## RFC Lifecycle

Draft → Submitted → Discussion → Decision → Accepted / Rejected / Deferred → Implementation → Released
