## 5) VitePress site in dedicated `website/` directory
- Create a `website/` directory at repository root to contain the VitePress site
- Initialize Node.js project within `website/`:
  ```bash
  cd website
  npm init -y
  ```
- Install VitePress as a development dependency:
  ```bash
  npm add -D vitepress@next
  ```
- Initialize the VitePress project (creates `.vitepress/` directory):
  ```bash
  npx vitepress init
  ```
- Configure `website/.vitepress/config.js` with Genealogix branding, Cloudflare domain `https://genealogix.io`, and use the `rewrites` option to map source directories from parent directory:
  - Map `../docs/guides/**` to `/guides/`
  - Map `../docs/development/**` to `/development/`
  - Map `../docs/examples/**` to `/examples/`
  - Map `../docs/quickstart.md` directly to `/quickstart`
  - Map `../specification/**` to `/specification/` (includes nested schema and rfcs)
  - Define navigation bar (nav) with links to Spec, Guides, CLI, Examples, RFCs, Schemas
  - Define sidebar (sidebar) with autogenerated sections for each category
  - Customize theme appearance and add Vue components as needed
- Test locally with dev server:
  ```bash
  npm run docs:dev
  ```
  (Site will be available at `http://localhost:5173`)
- Build for production:
  ```bash
  npm run docs:build
  ```

## 6) Cloudflare Pages setup
- Target project: `genealogix-docs`.
- Build command: `cd website && npm ci && npm run docs:build`.
- Output directory: `website/.vitepress/dist`.
- Configure custom domain `genealogix.io`, add DNS CNAME, enable HTTPS.
- Document any required adjustments for legacy repo-relative links in project docs or README.

## 7) Sanity checks
- Run `glx validate` across all examples (`docs/examples/`) and test fixtures (`glx/tests/`) after updates.
- Ensure Go tests still pass (`go test ./...`).
- Run `npm run docs:build` from `website/` directory to validate the static site output.
- Test the dev server locally (`npm run docs:dev` from `website/`) and spot-check navigation and sidebar structure.
