## Basic Workflow

### Individual Research

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐

│                 │    │    git add      │    │   git commit    │

│ Update citations│    │  sources/*.glx   │    └─────────────────┘
└─────────────────┘    └─────────────────┘            │
         │                      │                      │
         ▼                      ▼                      ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Validate      │    │   Test Locally  │    │  Push Changes   │
│                 │    │                 │    │                 │
│ glx validate    │    │ glx validate    │    │ git push        │

```

### Validation Commands
```bash
# Validate current directory
glx validate

# Validate specific directories
glx validate persons/
glx validate events/ places/

# Validate specific files
glx validate persons/person-john-smith.glx

# Full validation including examples
glx validate examples/complete-family/
```

## Branching Strategies

### Feature Branches (Recommended)

```
main (stable)
├── feature/add-john-smith-family
├── feature/add-leeds-census-data
├── feature/research-smith-occupation
└── feature/add-parish-records
```

**Feature Branch Workflow:**
```bash
# Create feature branch
git checkout -b feature/add-john-smith-family

# Make changes
# Edit persons/, events/, citations/
# Validate changes
glx validate

# Commit changes
git add .
git commit -m "Add John Smith family records

- Person: John Smith (1850-1920)
- Birth event with parish citation
- Leeds residence place record
- Marriage relationship to Mary Brown"

# Push and create pull request
git push origin feature/add-john-smith-family
```

### Research Topic Branches

```
main
├── research/1851-census-leeds
├── research/st-pauls-parish-1850s
├── research/smith-family-oral-history
└── research/victorian-leeds-blacksmiths
```

**Research Branch Workflow:**
```bash
# Create research branch for specific investigation
git checkout -b research/1851-census-leeds

# Add census data
# Create source, citations, person updates
glx validate

# Commit with detailed message
git commit -m "Add 1851 Census data for Smith family

Sources added:
- 1851 England Census (HO107, Piece 2319)

Persons updated:
- John Smith: occupation (blacksmith), residence
- Mary Smith: age, birthplace
- Added Thomas Smith (father)

Citations:
- Census household schedule references
- Quality rating: 2 (secondary source)"

# Merge findings to main when complete
git checkout main
git merge research/1851-census-leeds
```

## Collaboration Workflows

### Multiple Researchers

```
Researcher A:                    Researcher B:
├── feature/aunt-mary-letters    ├── feature/cousin-interviews
├── feature/civil-war-service    ├── feature/immigration-records
└── feature/photo-collection     └── feature/dna-matches
```

**Coordination:**
```bash
# Researcher A completes their work
git checkout main
git pull origin main  # Get latest changes
git checkout -b feature/aunt-mary-letters
# ... work on letters ...
git push origin feature/aunt-mary-letters

# Researcher B checks for conflicts
git checkout main
git pull origin main
git checkout -b feature/cousin-interviews
# ... work on interviews ...

# Merge when both are ready
git checkout main
git merge feature/aunt-mary-letters  # No conflicts
git merge feature/cousin-interviews  # May need resolution
```

### Evidence Integration

```
Evidence Types:
├── documents/ (birth certificates, letters)
├── records/ (census, parish registers)
├── photos/ (family portraits, houses)
├── oral/ (interview transcripts)
└── dna/ (test results, matches)
```

**Integration Workflow:**
```bash
# Start with documentary evidence
git checkout -b evidence-integration
git merge documents/birth-certificates
git merge records/census-data

# Validate integrated evidence
glx validate
# Check for conflicting assertions
glx check-conflicts  # Future feature

# Add corroborating evidence
git merge photos/family-portraits
git merge oral/interview-transcripts

# Final validation and commit
glx validate
git commit -m "Integrated evidence for Smith family 1850-1900

Combined sources:
- Birth certificates (primary)
- Census records (secondary)
- Family photos (supporting)
- Oral history interviews (tertiary)

Validated: All evidence chains complete, no conflicts detected"
```

## Merge Conflict Resolution

### File-Level Conflicts
When two researchers edit the same `.glx` file:

```bash
# Automatic merge fails
git merge feature/cousin-data
# CONFLICT: persons/person-john-smith.glx

# Manual resolution needed
# Edit persons/person-john-smith.glx to merge changes
# Validate merged result
glx validate persons/person-john-smith.glx

# Complete merge
git add persons/person-john-smith.glx
git commit -m "Merge cousin data for John Smith

Resolved conflicts:
- Combined birth date evidence
- Merged occupation information
- Added cousin relationship"
```

### Evidence Conflicts
When evidence suggests different conclusions:

```yaml
# Conflicting birth dates
# Researcher A: "1850-01-15" (from birth certificate)
# Researcher B: "1850-01-20" (from baptism record)

# Resolution strategy:
# 1. Keep both citations with quality ratings
# 2. Use assertion with confidence level
# 3. Add research note explaining discrepancy

assertions:
  assertion-birth-date:
    citations:
      - citation-birth-certificate  # Quality 3
      - citation-baptism-record     # Quality 3
    confidence: medium  # Multiple primary sources differ by 5 days
    notes: |
      Birth certificate shows January 15, baptism shows January 20.
      Possible explanations: delayed baptism, transcription error.
      More research needed to determine which is correct.
```

## Release Management

### Version Tags
```bash
# Create release after major milestone
git tag -a v1.0.0 -m "Release 1.0: Complete Smith family tree

Milestone: Complete evidence chain from 1850-1950
- All birth, marriage, death events documented
- Primary source citations for all major events
- Place hierarchy complete (England → Yorkshire → Leeds)
- Family relationships fully connected"

git push origin v1.0.0
```

### Archive Backups
```bash
# Create backup before major changes
git tag backup-before-census-integration
git push origin backup-before-census-integration

# After successful integration
git tag backup-after-evidence-review
git push origin backup-after-evidence-review
```

## Best Practices

### 1. Commit Message Format
```
Add [Person/Event/Place] [Name/ID] [brief description]

[Details]:
- What was added/changed
- Evidence sources
- Validation status]

Examples:
- "Add John Smith birth event with parish citation"
- "Update Mary Smith death date from death certificate"
- "Add Leeds place with coordinates and hierarchy"
```

### 2. Branch Naming
```
feature/[descriptive-name]     # New functionality
research/[topic-area]         # Research investigation
fix/[issue-description]       # Bug fixes
docs/[documentation-area]     # Documentation updates
```

### 3. Validation Before Commit
```bash
# Always validate before committing
glx validate

# Check specific areas if needed
glx validate persons/ relationships/
glx validate sources/ citations/

# Run full test suite for major changes
cd /path/to/genealogix-spec
glx validate examples/complete-family/
```

### 4. Evidence Integration
- Merge primary sources first
- Add supporting evidence as separate commits
- Resolve conflicts with research notes
- Validate complete evidence chains

This Git workflow ensures reliable collaboration while maintaining data integrity and complete audit trails for all genealogical research.
